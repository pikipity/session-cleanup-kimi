# Session Cleanup Skill 设计文档

## 项目概述

设计一个 Kimi Code CLI skill，用于安全清理历史会话，删除除当前会话外的其他所有 sessions。

**设计目标**：
- 保护当前会话不被误删
- 强制确认机制，避免误操作
- 跨平台兼容（macOS、Linux、Windows）
- 提供会话预览功能，帮助用户决策

---

## 核心原则

### 1. 当前会话保护
- 当前正在进行的会话**不可删除**
- 从删除列表中自动排除
- 如果用户指定删除当前会话，进入引导界面提供替代方案

### 2. 强制四步流程
无论用户如何触发，必须经过完整流程：
```
展示列表 → 选择操作 → 确认详情 → 执行删除
```

### 3. 动态引导
仅当用户明确指定删除当前会话时，才展示清理引导：
- `/clear` 清空当前会话
- 退出并新建会话
- 切换到其他会话
- [s] 跳过当前，继续删除其他

### 4. 跨平台兼容
- 纯 Python 实现，使用标准库
- `pathlib` 处理路径（自动适配 Windows/Unix）
- `shutil.rmtree` 跨平台删除
- UTF-8 编码统一

## 跨平台技术方案

### 路径处理
```python
from pathlib import Path
import os

def get_kimi_dir() -> Path:
    """跨平台获取 kimi 数据目录"""
    share_dir = os.environ.get("KIMI_SHARE_DIR")
    if share_dir:
        return Path(share_dir)
    return Path.home() / ".kimi"
```

### 路径哈希
```python
def get_workdir_hash(work_dir: str) -> str:
    """统一使用正斜杠格式计算 MD5，确保跨平台哈希一致"""
    normalized = Path(work_dir).as_posix()
    return hashlib.md5(normalized.encode()).hexdigest()
```

### 文件操作
```python
# 读取：统一 UTF-8，显式指定 newline
with open(file_path, 'r', encoding='utf-8', newline='') as f:
    ...

# 删除：使用 shutil.rmtree（跨平台）
import shutil
shutil.rmtree(session_path)
```

### 平台差异处理

| 场景 | Windows | macOS/Linux |
|-----|---------|-------------|
| 文件占用 | 常见，需处理 PermissionError | 较少见 |
| 退出快捷键 | Ctrl+Z+Enter | Ctrl+D |
| 家目录 | C:\Users\<user> | /Users/<user> 或 /home/<user> |

## 脚本功能说明

### list_sessions.py
- 扫描 `~/.kimi/sessions/` 下所有工作目录
- 读取 `context.jsonl` 统计消息数
- 读取 `wire.jsonl` 获取时间和标题
- 识别当前会话（环境变量 → 当前目录最新 → 全局最新）
- 输出 JSON 格式，含全局索引

### get_preview.py
- 接收 `--indices` 参数
- 读取指定 sessions 的最后 2 条消息
- 格式化输出（角色、时间、内容截断）

### delete_sessions.py
- 接收 `--indices` 或 `--all-except` 参数
- **自动跳过当前会话**（双重保护）
- 使用 `shutil.rmtree` 跨平台删除
- 处理 Windows 文件占用错误
- 返回删除结果 JSON

### 测试修复 1：添加 path 字段
**问题：** get_preview.py 和 delete_sessions.py 无法找到会话路径
**原因：** list_sessions.py 输出时丢失了 `path` 字段
**修复：** 在 `sessions_list.append()` 中添加 `"path": s["path"]`
**文件：** `scripts/list_sessions.py`

### 测试修复 2：处理 list 类型 content
**问题：** `content.strip()` 报错 `'list' object has no attribute 'strip'`
**原因：** kimi-cli 的 context.jsonl 中，`content` 有时是 list（如工具调用结果）
**修复：** 添加类型检查，list 类型显示 "[复杂内容]"
**文件：** 
- `scripts/list_sessions.py` - `get_session_title()` 和 `get_last_preview()`
- `scripts/get_preview.py` - `get_last_messages()`

### 测试修复 3：清理空工作目录
**问题：** 删除会话后，工作目录的 hash 文件夹残留
**原因：** 只删除了 `<hash>/<session-id>/`，没删 `<hash>/`
**修复：** 
1. 删除会话后检查父目录是否为空
2. 过滤 `.DS_Store` 文件判断
3. 空目录一并删除
**文件：** `scripts/delete_sessions.py` - `delete_session()` 函数
