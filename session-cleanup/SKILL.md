# Safe Session Cleanup

安全清理 Kimi CLI 会话，**当前会话始终受保护**。

## 核心原则

1. **当前会话保护**：当前会话不可删除，自动排除
2. **强制四步流程**：展示 → 选择 → 确认 → 执行
3. **动态引导**：指定删除当前会话时，提供替代方案（/clear、新建会话等）
4. **跨平台兼容**：macOS、Linux、Windows

## 执行流程

### 第一步：列出所有会话

执行脚本获取会话列表：

```bash
python scripts/list_sessions.py
```

向用户展示：
- 所有会话按工作目录分组
- 全局索引（1-N）
- 消息数、大小、最后消息预览
- 当前会话标记 💬

### 第二步：选择操作

提供选项：
- `[1]` 删除其他所有会话
- `[2]` 保留各目录最新，删除其他
- `[3]` 仅清理当前目录的其他
- `[4]` 删除指定编号（用户输入如 "2 4 5"）
- `[q]` 取消

**处理选项 4（指定编号）：**

如果用户输入包含当前会话索引（如 "1 2 3"），展示引导界面：

```
⚠️ 无法删除当前会话 [1]

如需重新开始：
  [1] 使用 /clear 清空当前会话（推荐）
  [2] 退出并新建会话
  [3] 切换到其他会话
  [s] 跳过当前，继续删除其他指定的
  [n] 返回
  [q] 取消
```

- 用户选 `[1][2][3]`：展示操作说明，按 Enter 返回
- 用户选 `[s]`：排除当前，进入确认界面
- 用户选 `[n]`：返回选择菜单
- 用户选 `[q]`：退出

### 第三步：确认界面

展示将要删除和保留的列表：

```
❌ 将删除 X 个会话：
   [索引] 时间 标题  消息/大小/路径  最后预览

✅ 将保留 Y 个会话：
   [索引] 时间 标题  消息/大小/路径

请选择:
  [y] 确认删除
  [n] 返回重新选择
  [p] 预览要删除的会话
  [q] 取消退出
```

**预览功能（按 p）：**

执行 `get_preview.py --indices <要删除的>`，展示每个会话的最后 2 条消息。

### 第四步：执行删除

用户确认后执行：

```bash
python scripts/delete_sessions.py --indices 2 3 4
```

脚本自动跳过当前会话（双重保护）。

报告结果：
```
✅ 已删除 3 个会话：
   - [2] 代码审查 (89 KB, 18 条)
   - [3] API 设计 (320 KB, 42 条)
   ...

💬 保留当前会话：
   - [1] Skill 开发讨论 (156 KB, 24 条)
```

## 快捷指令处理

| 用户说法 | 处理 |
|---------|------|
| "清理 session" | 展示列表和选项 |
| "删除 2 和 3" | 预选选项 4 填入 "2 3"，仍需确认 |
| "清理其他" | 预选选项 1，仍需确认 |

**所有指令都必须经过确认界面，禁止自动执行。**

## 跨平台说明

- 路径：`~/.kimi/`（macOS/Linux）或 `%USERPROFILE%\.kimi\`（Windows）
- 删除：`shutil.rmtree`（跨平台）
- 编码：UTF-8

## 故障处理

| 问题 | 处理 |
|-----|------|
| Windows 文件占用 | 提示"文件被占用，请关闭后重试" |
| 无效编号 | 提示"编号 X 不存在，请重新输入" |
| 无其他会话 | 提示"只有一个当前会话，无需清理" |
